Blocs, Procs, and Lambdas

A bloc is an anonymous block of code for example...

["kazoozles", "orange gummies", "orange tictacs"].each { |yum| p "I heart " + yum + "!" } 
                    # =>"I heart kazoozles!"
                    # =>"I heart orange gummies!"
                    # =>"I heart orange tictacs!"

The bloc here would be the portion in between the curly braces { |yum| p "I heart " + yum + "!" } or you can also use the do..end syntax.

A proc is a bloc with a name, so you can re-use the code without having to rewrite the code each time. They're also objects unlike Blocs so they have all the functionality of an object.

              
                    steaks = ["Ribeye", "T-Bone", "Porterhouse"]
                    candies = ["kazoozles", "orange gummies", "orange tictacs"]

                    yummies = Proc.new { |yum| p "I heart " + yum + "!" }

                    p steaks.each(&yummies)
                    # =>"I heart Ribeye!"
                    # =>"I heart T-Bone!"
                    # =>"I heart Porterhouse!"
                    
                    p candies.each(&yummies)
                    # =>"I heart kazoozles!"
                    # =>"I heart orange gummies!"
                    # =>"I heart orange tictacs!"

A lambda is essentially a proc that behaves like a method. It'll check to see if you've passed the correct number of arguments, if for some reason you're short or extra an argument or two lambda will throw an error. Procs will go ahead and run your code regardless. Lambda's and procs also handle returns differently. For example...

              def steak_proc
                proc = Proc.new { return "I heart ribeyes" }
                proc.call
                "Procs do what they want"
              end

              p steak_proc # => I heart ribeyes

              def steak_lambda
                lambda = lambda { return "I heart ribeyes" }
                lambda.call
                "Lambdas do what I want"
              end

              puts steak_lambda # => Lambdas do what I want

From the example you can see proc returns itself and doesn't bother running the rest of steak_proc. A lambda in contrast will run itself and run the rest of steak_lambda method.
